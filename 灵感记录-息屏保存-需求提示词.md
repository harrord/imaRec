# imaRec 灵感记录：息屏自动保存功能 — 需求提示词

> **范围**：本次修改仅新增「息屏保存灵感」功能，原有功能（双击进入灵感模式、单击按钮保存灵感、普通分段、暂停、上传等）全部保留且不受影响。


## 一、需求概述

在现有灵感记录功能的基础上，新增一种更自然的结束方式：

- **进入灵感模式后**：App 保持屏幕常亮，防止因超时自动熄屏而意外中断灵感。
- **用户想结束灵感时**：按下电源键将屏幕熄灭，App 监测到熄屏事件后，自动将当前灵感片段落盘、上传到灵感目标文件夹，并回到普通录音模式。
- **原有手动保存仍然保留**：在灵感模式期间，用户继续单击锁屏控件上的分段按钮，依然可以手动保存灵感。

---

## 二、技术方案

### 2.1 灵感模式期间保持屏幕常亮（防自动熄屏）

在 `SegmentController` 中新增一个独立的 `WakeLock`，类型为 `PowerManager.SCREEN_DIM_WAKE_LOCK`：

- **进入灵感模式时（`enterInspirationMode`）**：获取此 WakeLock。
- **退出灵感模式时（`saveInspirationSegment`）**：释放此 WakeLock。
- **兜底清理（`stopSession`）**：若会话直接结束，确保该 WakeLock 也被释放，防止泄漏。

**说明**：`SCREEN_DIM_WAKE_LOCK` 阻止系统自动熄屏，但**不阻止用户手动按电源键熄屏**——这正是我们想要的。

### 2.2 监听屏幕熄灭事件（`ACTION_SCREEN_OFF`）

在 `RecordingService` 中动态注册一个 `BroadcastReceiver`：

- **注册时机**：录音会话开始时（`startRecording`）。
- **注销时机**：`stopForegroundSafely()` 和 `onDestroy()` 中。
- **Receiver 行为**：收到 `Intent.ACTION_SCREEN_OFF` 时，调用 `SegmentController.stopInspirationByScreenOff()`。

### 2.3 SegmentController 新增公开方法

```kotlin
/**
 * 通过屏幕熄灭自动结束灵感记录。
 * 仅在当前处于灵感模式时生效；普通模式下忽略。
 */
fun stopInspirationByScreenOff()
```

**内部实现**：

1. 若 `inspirationMode == false`，直接返回。
2. 调用现有的 `saveInspirationSegment()` 完成落盘、上传、切回普通模式、开新段继续录音。
3. `saveInspirationSegment()` 内部会自动释放 `inspirationWakeLock`。

## 四、交互流程

### 4.1 正常灵感记录 → 息屏保存

```
用户双击分段按钮
  → 进入灵感模式
    → 开新段继续录音
    → 获取 SCREEN_DIM_WAKE_LOCK（屏幕不会自动熄灭）
    → 锁屏控件反馈行显示："灵感开始记录，按电源键保存"
      → 用户持续说话（灵感输出）
        → 用户按下电源键
          → 屏幕熄灭，系统发送 ACTION_SCREEN_OFF
            → RecordingService.Receiver 收到广播
              → controller.stopInspirationByScreenOff()
                → 检查 inspirationMode == true
                  → 调用 saveInspirationSegment()
                    → 释放 inspirationWakeLock
                    → 落盘当前片段并上传到灵感文件夹
                    → 新建并选中灵感文件夹 Tab
                    → 开新段继续普通录音
                    → 锁屏控件显示："灵感已保存到「xxx」"（5 秒后清除）
```

### 4.2 灵感记录 → 手动按钮保存（原有功能保留）

```
用户双击分段按钮进入灵感模式
  → 屏幕保持常亮
    → 用户单击分段按钮（手动保存）
      → 调用 controller.manualSegment()
        → 灵感模式内任何点击都触发 saveInspirationSegment()
          → 释放 WakeLock、落盘、上传、回到普通模式
```

### 4.3 普通录音（未进入灵感模式）

```
录音中，未进入灵感模式
  → 系统正常超时熄屏
    → ACTION_SCREEN_OFF 广播照样发出
      → controller.stopInspirationByScreenOff() 检测到 inspirationMode == false
        → 直接忽略，不做任何操作
          → 普通录音不受影响
```

## 五、边界情况与处理

| 场景 | 预期行为 |
|------|----------|
| 灵感模式期间，系统自动超时熄屏 | **不会发生**。因为 `SCREEN_DIM_WAKE_LOCK` 阻止了自动熄屏。 |
| 灵感模式期间，用户误按电源键 | 屏幕熄灭，灵感被保存。属于预期行为，副作用可控。 |
| 用户手动点击按钮保存灵感后，再按电源键 | 此时 `inspirationMode == false`，熄屏广播被忽略，不影响普通录音。 |
| 灵感模式期间收到来电导致屏幕变化 | 来电亮屏/熄屏可能触发 `ACTION_SCREEN_OFF`，会保存灵感。此情况极少见，且保存灵感无副作用。 |
| 录音会话直接停止（未保存灵感） | `stopSession()` 兜底释放 `inspirationWakeLock`，不会泄漏。 |

## 六、权限说明

无需新增任何权限。现有 `AndroidManifest.xml` 已声明：

```xml
<uses-permission android:name="android.permission.WAKE_LOCK" />
```

`ACTION_SCREEN_OFF` 广播的接收也无需额外权限。

## 七、备注

1. **原有功能全部保留**：
   - 双击分段按钮进入灵感模式。
   - 灵感模式下单击按钮保存灵感。
   - 普通分段、分组切换、暂停/继续、定时停止、自动分段等逻辑完全不变。

2. **WakeLock 拆分**：
   - `PARTIAL_WAKE_LOCK`（已有）：保证后台录音 CPU 不休眠。
   - `SCREEN_DIM_WAKE_LOCK`（新增）：仅灵感模式期间持有，保证屏幕不自动熄灭。
   - 两者独立管理，互不影响。

3. **动态 Receiver**：
   - 仅在录音会话活跃期间注册，避免全局监听带来的不必要开销。
   - 会话结束（`Idle`）时立即注销。
